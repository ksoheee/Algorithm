### 문제 설명

<p>LCS(Longest Common Subsequence, 최장 공통 부분 수열)문제는 두 수열이 주어졌을 때, 모두의 부분 수열이 되는 수열 중 가장 긴 것을 찾는 문제이다.</p>

<p>예를 들어, ACAYKP와 CAPCAK의 LCS는 ACAK가 된다.</p>

### 입력 

 <p>첫째 줄과 둘째 줄에 두 문자열이 주어진다. 문자열은 알파벳 대문자로만 이루어져 있으며, 최대 1000글자로 이루어져 있다.</p>

### 출력 

 <p>첫째 줄에 입력으로 주어진 두 문자열의 LCS의 길이를 출력한다.</p>

### 풀이
<pre>처음에는 루프를 돌리면서 뒤의 문자열이 앞의 문자열과 같을때 카운트를 세는 방식으로 구현했는데, 
이렇게 되면 한 문자열을 기준으로 LCS를 만들기 떄문에 가장 긴 공통 수열이 보장되지 않는다
LCS는 매 위치에서 문자를 쓰거나, 버리거나 두가지 선택이 존재하므로 그리디는 아니고
같은 상태를 계속 반복하므로 dp를 사용해야 한다

<b>상태</b>
dp[i][j] = s1을 i번째 문자까지 봤을 때, s2를 j번째 문자까지 봤을 때 지금까지 만들 수 있는 최대 공통 부분 수열 길이

<b>점화식</b>
문자가 같으면 -> dp[][]+1(문자사용)
다르면 -> 문자열 둘 중 문자 하나 버리는 두 경우 중 최대값

<b>구현</b>
같을때는 +1해서 추가해 주면 되고, 다르다면 else 로 그전에 저장되어있는 최대값을 저장해주면된다.  
문자가 다르다면 문자를 다 버리고 세는게 아니고 다른 문자를 버리고 세면 되는 것 이기 때문에 문자가 다를때는 else문에서 자동으로 걸러준다.</pre>
