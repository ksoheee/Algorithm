### 풀이 - 완전탐색, DFS
<p>할인율을 조합하는 함수, 서비스가입여부와 구매비용을 계산하는 함수가 필요하다.</p>
<h4>할인율 조합 함수</h4>
<p>완전 탐색을 위해서는 4^이모티콘 갯수(m) 만큼의 조합이 필요하다.
n번째의 이모티콘 할인율을 하나 정한 후 n+1번째로, n+2로 m번 만큼 깊이 탐색했다가 끝나면 돌아와서 할인율의 갯수인 4번을 또 반복하여 할인율을 정한다. 
종료 조건은 이모티콘 갯수만큼 할인율을 정했을때 -> 구매비용 계산하는 함수 호출한다.</p>
<p>=> DFS는 트리 구조 처럼 동작해서 각 이모티콘 마다 할인율(4가지)을 하나 선택하고 다음 이모티콘으로 이동한다. 모든 이모티콘에 할인율을 적용했을 때 리프 노드에 도달하고, 그 시점에 구매비용, 가입여부확인 함수를 호출한다.</p>
<h4>구매비용과 서바스가입 확인 함수</h4>
<p>사용자의 수 만큼 반복하게 되는데, 각각의 사용자가 이모티콘 개수(m)만큼 할인율을 적용하여 구매한다.
구매금액이 사용자가 지정한 금액보다 크면 서비스에 가입하고, 그렇지 않으면 금액에 추가 시킨다.</p>
<p>=>각 사용자에 대해 이모티콘들을 확인하며, 할인율이 사용자 최소기준이상이면 구매 금액에 합산한다. 총합이 사용자의 기준 가격 이상이면 이모티콘 구매를 취소하고 플러스 가입 처리한다.</p>
<h4>최적 결과 갱신</h4>
<p>사용자의 수 만큼 확인을 했다면 서비스 가입수가 최대 가입수인지, 최대 금액인지 순서대로 확인하고 값을 갱신한다.</p>
<p>=> 각 조합 (가입자 수, 매출액)을 계산한뒤 현재 최적값과 비교해서 더 좋은 경우라면 전역변수에 갱신한다.</p>
<p>함수는 자기 일만 할 수 있도록, 외부 상태(전역 변수 등)을 바꿔버리는 일을 하지 않도록 바꾸어주었다.</p>
